### 问卷反向设计流程

```
MD 评估标准定好
    ↓ 提取所有判断维度
每个维度设计一个问卷问题
    ↓
每个选项直接对应一个评估档位（满足/部分满足/不满足）
    ↓ 产出
映射表：问卷问题 → MD 检查项 → 评估标准 → 选项 → 档位
```

### 示例

```
MD 评估标准：
  满足 = 已有碳审计报告 + 按季度申报
  部分满足 = 有能耗数据但未区分排放类型
  不满足 = 无任何数据

↓ 反推问卷 ↓

Q: 你们有碳排放相关数据吗？
  A) 有完整碳审计报告，按季度申报   → 满足
  B) 有能耗/排放数据，但不完整       → 部分满足
  C) 没有                           → 不满足
```

### 约束式 Prompt 结构

```
【规则】4 条约束（只能匹配，不能编）
【示例】1 个手写完整 Few-shot（锚定输出质量）
【法规资料】从 MD 文件加载的内容
【问卷答案】用户的回答
→ 输出结构化 JSON
```

### 评分聚合公式

```python
item_score = {"green": 100, "yellow": 50, "red": 0}

e_score = avg([item_score[item.status] for item in E_items])
s_score = avg([item_score[item.status] for item in S_items])
g_score = avg([item_score[item.status] for item in G_items])

total = e_score * 0.3 + s_score * 0.35 + g_score * 0.35
```

### 相同输入缓存

```python
cache_key = hash(answers + country + standard)
if cache_key in memory_cache:
    return cached_result
```

### 改善路线图

按**改善难度 × 法规重要性**排序：

```
第 1 步（1-2周，容易）：
  劳工权益 → 完善员工合同（只需修改模板文件）

第 2 步（1-2月，中等）：
  废物管理 → 制定废物分类方案（需联系供应商）

第 3 步（3-6月，困难）：
  碳排放报告 → 建立完整碳审计体系（需外部顾问）
```

排序逻辑：红灯优先于黄灯，容易的优先于困难的。
